    1. Carga de los componentes de la página.

    La carga se realiza de arriba a derecha, y “primero” el documento, después “recursos”.
    ¿Cómo se Cargar una página?:
    + Cuando se abre por primera vez una página.
    + Al recargar/refrescar la página.
    + Cuando se regresa después de irse a otra página (enlace, formulario, etc.,).

    Load

    El evento load .Ocurre cuando se terminan de cargarse distintos elementos de una página (window, document, body, frame, iframe, img, link, script, style).
    + Este evento no se produce cuando la página se carga desde memoria caché.
    + Utilizar cuando un script necesite que una página web haya cargado completamente todo el contenido (incluidas imágenes, archivos script, CSS, etc.).

    DOMContentLoaded

    El evento DOMContentLoaded es disparado cuando el documento HTML ha sido completamente cargado, sin esperar hojas de estilo, images y subframes para finalizar la
    carga, es decir el documento ha terminado de cargar (pero no sus recursos dependientes).
    + Se puede utilizar para insertar o modificar el DOM, y cualquier otra acción antes de terminar de cargar todo los recursos de la página.

    readyState

    Propiedad de document que describe el estado de carga del documento.
    +Los posibles valores son:
    -“loading” cargando: El documento todavía se está cargando.
    -“interactive” interactivo: El documento ha terminado de cargarse y el documento se ha analizado, pero los sub-recursos como imágenes, hojas de estilo y marcos
    todavía se están cargando.
    +En esta fase, ya podemos acceder a los elementos DOM, pero, generalmente no al resto de recursos.
    +Se produce a “continuación” el evento DOMContentLoaded.
    -“complete” Completado: El documento y todos los sub-recursos han terminado de cargar.
    + Este estado indica que el evento load está a punto de dispararse.
    +Cuando cambia el valor de esta propiedad, se dispara el evento readystatechange en el objeto document.

    readystatechange

    El evento readystatechange se dispara en el objeto document, cuando la propiedad document.readyState, cambia de valor.
    + Este evento es una alternativa a los eventos DOMContentLoaded (interactive ) y load (complete).
    + La propiedades más características de este evento son: type, target, bubbles y cancelable.

    2. Carga de la página.
    pageshow

    El evento Pageshow ocurre cuando se visualiza una página web.
    -El evento pageshow, a diferencia del evento load, se produce cada vez que se carga o recarga la página.

    resize

    El evento resize se produce cuando se redimensiona la ventana del navegador, o el frame, en caso de que la página los tenga.
    -Se produce de forma ininterrumpida hasta que deja de dimensionar.

    scroll

    El evento scroll ocurre cuando varía la posición de la barra de desplazamiento (scroll) de un elemento(window,body, div, form, pre,.. ). También cuando se
    mueve la rueda de ratón.
    -Al igual que resize se produce de “forma continuada”.
    -Nota: use la propiedad de estilo overflow CSS para crear una barra de desplazamiento para un elemento.

    hashchange

    El evento hashchange hashchange ocurre cuando se cambia la parte de la URL que comienza con el símbolo '#'.
    Este evento se invoca:
    -Al cambiar la parte del anclaje con las propiedades: location.hash o location.href del objeto Location.
    -Navega a la página actual con un marcador diferente (usa los botones "atrás" o "avanzar").
    -Hacer clic en un enlace a un marcador de marcador.

    3. Descargando la página.

    ¿Cómo se Descargar una página?: + Cerrando la ventana del navegador.
    + Al recargar/refrescar la página.
    + Se abandona/se navega fuera de la página:
    -Haciendo clic en un enlace que nos lleve a otra página.
    -Enviando un formulario.

    beforeunload

    El evento beforeunload ocurre cuando el documento está a punto de ser descargado.
    + Al producirse este evento, el documento todavía es visible y la descarga de la página se puede cancelar solo en este punto
    + Este evento, permite realizar acciones (mostrar un mensaje en un cuadro de diálogo de confirmación para informar al usuario si desea quedarse o abandonar la
    página actual) antes de descargar completamente (unload) la página.
    Para “cancelar” disponemos de:
    El método de evento: preventDefault().
    La propiedad de evento: returnValue.
    O enviado como return una cadena distinta
    +Por defecto se visualiza un mensaje predeterminado que aparece en el cuadro de confirmación es diferente en diferentes navegadores. Sin embargo, el mensaje
    estándar es algo así como "¿Seguro que quieres salir de esta página?". Este mensaje no puede ser eliminado.
    -Se puede escribir un mensaje personalizado junto con el mensaje predeterminado.

    A partir del 25 de Mayo del 2011, la especificación HTML5 establece que llamadas a los métodos window.alert(), window.confirm(), y window.prompt() pueden ser
    ignoradas durante este evento.
    +Varios navegadores ignoran el resultado del evento y no le preguntan al usuario por confirmación en absoluto. El documento siempre se descargara
    automáticamente.
    pagehide

    El evento pagehide ocurre cuando deja visualizarse la página.

    unload

    El evento unload ocurre cuando se descarga completamente distintos elementos de una página.
